# Data

Patterns for managing data consistency and state across services.

| Pattern | Description |
|---------|-------------|
| [Outbox](./outbox) | Transactional outbox for reliable message publishing |
| [CQRS](./cqrs) | Command Query Responsibility Segregation — separate read/write models |
| [Saga](./saga) | Orchestrated long-running transactions with compensating actions |
| [Saga Choreography](./saga-choreography) | Event-driven saga with decentralised coordination |
| [Event Sourcing](./event-sourcing) | Append-only event log as the source of truth |
